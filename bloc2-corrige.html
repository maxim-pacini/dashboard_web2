TODO 1 — Navigation par clic : les 5 étapes

L'idée centrale : chaque <li> porte data-page="homePage". jQuery lit cet attribut pour savoir quelle section afficher.

$(".navLink").on("click", function() { // ÉTAPE A — Reset de l'état actif $(".navLink").removeClass("activeNav"); $(this).addClass("activeNav"); // ÉTAPE B — Lire l'ID de la page cible let page = $(this).data("page"); // ÉTAPE C — Cacher toutes les sections $(".page").removeClass("activePage").hide(); // ÉTAPE D — Afficher la bonne section $("#" + page).fadeIn(400).addClass("activePage"); // ÉTAPE E — Toast de notification $("#toast") .text("→ " + page) .show() .delay(1800) .fadeOut(400); });

Erreur fréquente : appeler .hide() ET .removeClass("activePage") sans .show() ensuite → la page ne réapparaît jamais. Toujours faire le couple hide → fadeIn.


Autre erreur : oublier le # dans "#" + page. Sans le #, jQuery cherche une balise HTML (ex: homepage) au lieu d'un ID.

TODO 2 — Toggle Menu avec .animate()

La valeur "toggle" dans .animate({ width: "toggle" }) est un raccourci jQuery : si la largeur est > 0, il l'anime vers 0. Sinon, il la restaure à sa valeur d'origine.

$("#toggleMenu").on("click", function() { $("#sidebar").animate({ width: "toggle" }, 400); });

Alternative : on peut tester l'état visible/caché avec .is(":visible") et choisir manuellement :

if ($("#sidebar").is(":visible")) { $("#sidebar").animate({ width: "0" }, 400); } else { $("#sidebar").animate({ width: "240px" }, 400); }

Les deux approches fonctionnent. "toggle" est plus concis, mais la version if/else est plus explicite pour comprendre ce qui se passe.

.data() — Lire les attributs data-*

En HTML5, on peut stocker des données personnalisées sur n'importe quel élément avec des attributs data-*.

<!-- HTML --> <li data-page="homePage" data-color="rouge">...</li> // jQuery : lire $(this).data("page") // → "homePage" $(this).data("color") // → "rouge" // Équivalent natif JS (moins pratique) : this.getAttribute("data-page") // → "homePage"

Avantage de .data() : jQuery convertit automatiquement le type. data-target="42" retourne le nombre 42, pas la chaîne "42".

BONUS — Barre de progression avec Set

Un Set JavaScript est une collection de valeurs uniques. On l'utilise ici pour ne compter chaque page visitée qu'une seule fois.

// Déclarer au début du script const visitedPages = new Set(["homePage"]); // Dans le clic .navLink, ajouter la page visitée visitedPages.add(page); // add() ignore les doublons // Calculer le pourcentage const pct = visitedPages.size / 5 * 100; // Animer la barre $("#progressFill").animate({ width: pct + "%" }, 400);

Set.size = nombre d'éléments uniques. Si on visite 3 pages différentes : size = 3, donc 3/5*100 = 60%.
